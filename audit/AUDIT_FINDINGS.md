# NeuroAD — Four-Domain Audit Findings


## Alzheimer's structural-MRI neuroimaging ML (foundation-model embedding auditing / batch-effect leakage)

**Summary:** The engineering is genuinely careful (subject-disjoint CV, CI-gated anchors, honest "leakage margin" framing, explicit prior-art positioning) and the core scientific premise — frozen brain-FM embeddings leak scanner/site and must be gated against — is real and well-supported. But the two hero results do not cleanly survive the tool's own gauntlet: the real OASIS "survivor" collapses to chance on the brain-age star, and the promoted synthetic "strong candidate" barely passes the leakage star and is rescued almost entirely by a fabricated p-tau217 correlation. The biomarker-anchor step — the advertised differentiator ("Proteins confirm it") — is 100% synthetic and never run on real molecular data, the brain-age control is over-aggressive in a way an AD-imaging expert would flag, the load-bearing prior-art citations are unverifiable/future-dated, and several shipped artifacts contradict the docs. Fixable before the deadline, but the demo should stop leading with beats that fail the referee.

**Strengths:**
- Core premise is scientifically sound and correctly scoped: frozen FM embeddings leak scanner/site is a real, documented phenomenon, and the project honestly claims productization (a runnable referee) rather than discovery of the effect.
- Real, demonstrable leakage result: on 3,984 healthy OpenBHB brains with no disease, structural features predict scanner field strength at AUC 0.89 (and the frozen Neuro-JEPA 768-d embedding at ~0.93 PCA-10) — a genuine, honestly-caveated batch-effect demonstration.
- Methodologically literate: subject-disjoint cross-validated AUC, Fisher-z CI-gated biomarker anchor, Wilson/Cohen's-d cluster stats, p>>n overfitting explicitly acknowledged (raw 0.998 vs PCA-10 0.93), and a conservative-lower-bound argument for the leakage-margin CV asymmetry.
- Unusually honest self-documentation: DATA_PROVENANCE.md openly states the demo over-indexes on synthetic and names it as the credibility risk; caveats are surfaced on every card.

**Findings:**
- [high] **Both hero 'survivors' fail a heavy-weight star test — the real one collapses on brain-age, the promoted one leans on synthetic biomarkers** — reports/oasis.json (the real AD-vs-CN 'survivor') FAILS the brain-age star: effect retained 1% after control (AUC 0.82→0.50, chance), so the tool's own gauntlet says the real signal is indistinguishable from generic aging. The promoted synthetic 'strong candidate' (synthetic_survivor.json, score 88) has a leakage margin of only +0.04 (star WEAKENED) — its own judge_reasoning calls the artifact reading 'the more defensible one' — yet it is rescued to 'strong candidate' almost entirely by a fabricated p-tau217 r=0.40. Neither hero cleanly passes both starred tests.
    - evidence: reports/oasis.json brain_age retained 0.012, promoted:false; reports/synthetic_survivor.json margin +0.04, ptau217_r 0.404, judge_reasoning
- [high] **The biomarker-anchor differentiator ('Proteins confirm it') is entirely synthetic and never validated on real molecular data** — The p-tau217 r≈0.43 / GFAP r≈0.35 anchor values are calibration targets in calibration.CAL chosen to sit inside a literature range, not measurements — no open MRI+plasma cohort is used, as DATA_PROVENANCE.md concedes. This anchor is the hard promotion gate (scoring.py), so the promotion of the emotional-core SURVIVOR rests on a fabricated correlation, and the tagline 'Proteins confirm it' overstates what is demonstrated.
    - evidence: src/neuroad/calibration.py CAL['ptau217_r']; docs/DATA_PROVENANCE.md 'no open dataset'; scoring.py HARD GATE
- [medium] **Brain-age control conflates disease atrophy with aging, making the star over-aggressive and its 'kill' uninterpretable** — gauntlet.test_brain_age regresses out *predicted brain age* (not the brain-age gap) from the embedding, then re-probes. On OASIS the 'embedding' is 3 collinear volumetric features (nWBV/eTIV/ASF); AD atrophy loads on the same direction as brain-age, so this control removes the disease signal by construction (hence retained 1%). It cannot separate 'accelerated aging' from 'AD-specific atrophy' — an AD-imaging expert would call this hand-wavy.
    - evidence: src/neuroad/gauntlet.py lines 161-177; reports/oasis.json brain_age R2=0.57 retained 0.012
- [medium] **Load-bearing prior-art citations are unverifiable and future-dated; the whole 'we don't overclaim' defense rests on them** — The positioning ('published prior art, we productize') hinges on arXiv:2604.14441, arXiv:2606.09189, Neuro-JEPA arXiv:2606.14957, and 'PathoROB, Nature Communications 2026' — all 2026, none verifiable offline. If a judge Googles one and it does not resolve, the entire honesty framing collapses. These cites must each be resolved to a real DOI/abstract or removed.
    - evidence: src/neuroad/calibration.py PRIOR_ART; README.md lines 18-27
- [medium] **Docs contradict shipped artifacts, and the flagship report mislabels 3 morphometric features as Neuro-JEPA embeddings** — reports/oasis.json (n=263) is labeled substrate 'frozen Neuro-JEPA structural embeddings', but the real Neuro-JEPA OASIS run is n=61 (oasis_neurojepa_ad.json) — the n=263 run is the 3-feature tabular block, so the flagship real survivor overstates its substrate. DATA_ACCESS.md still says the Neuro-JEPA path is 'documented, not shipped' and 'nothing depends on it', while README/PROVENANCE/reports ship it as a headline; and DATA_PROVENANCE calls OASIS 'PROMOTED' while oasis.json says promoted:false, score 69.
    - evidence: reports/oasis.json vs reports/oasis_neurojepa_ad.json; docs/DATA_ACCESS.md 'documented, not shipped' vs docs/DATA_PROVENANCE.md 'PROMOTED'
- [low] **Gated CC-BY-NC-ND Neuro-JEPA weight provenance is unexplained given the auto-deny-gmail note** — Real 768-d Neuro-JEPA embeddings are shipped (data/real/*_neurojepa_embeddings.csv), yet DATA_ACCESS.md states the weights are 'HF-gated behind an institutional-PI agreement that auto-denies personal/gmail emails' and the assistant 'cannot accept such an agreement' — while the operating account is a gmail address. Frozen-inference-only + git-ignored embeddings is a defensible non-derivative stance, but the access path should be stated explicitly to avoid a license/provenance question.
    - evidence: data/real/oasis1_neurojepa_embeddings.csv (773 cols); docs/DATA_ACCESS.md 'auto-denies personal / gmail emails'

**Enhancements:**
- [L] **Lead the demo with a REAL survivor that passes both stars — add a second open AD cohort** — Add MIRIAD (open, self-serve, 46 AD/23 CN) or push the OASIS-3 1-week DUA so the leakage star is a real multi-scanner test (not the OASIS1-vs-OASIS2 cohort reframe) AND run brain-age with a control the AD signal can survive. A real disease finding that clears both stars is the single biggest credibility lift.
- [M] **Report brain-age adjustment two ways and add the standard scanner control** — Show both the predicted-brain-age (aggressive) and brain-age-GAP (Franke/Gaser standard) adjustments so a reviewer sees the survivor isn't killed by one over-aggressive residualization, and add ComBat/ComBat-GAM harmonization as the cited-standard scanner control rather than only LDA direction-scrubbing.
- [M] **Badge every biomarker number as SYNTHETIC and pull it from the headline until real** — Since the p-tau217/GFAP anchor is calibrated, not measured, remove 'Proteins confirm it' from the top-line pitch and visibly badge the anchor in the UI/reports; optionally demonstrate the anchor mechanic on a real reachable proxy (amyloid-PET Centiloid or CSF in OASIS-3) to make the gate real.
- [S] **Resolve or delete every prior-art citation and hyperlink DOIs in-repo** — Verify arXiv:2604.14441, 2606.09189, 2606.14957 and PathoROB against live abstracts; embed resolvable links in calibration.PRIOR_ART and README. Any cite that does not resolve must be removed — the referee's credibility cannot rest on an unverifiable reference.
- [M] **Make the leakage-margin comparison symmetric and report CIs** — Add a leave-one-site-out (site-disjoint) scanner AUC alongside the standard-CV one so the margin isn't an asymmetric comparison a reviewer can attack, and report bootstrap CIs on the n=96 Neuro-JEPA leakage numbers where p>>n makes point estimates fragile.
- [S] **Regenerate all reports and reconcile docs with artifacts** — Re-run every reports/*.json under the current scoring logic (fix the OASIS promoted flag and the n=263 'Neuro-JEPA' substrate mislabel), and update DATA_ACCESS.md's 'documented, not shipped' section now that embeddings ARE shipped, so no doc contradicts a shipped file.

## Data Science / Statistics methodology audit — NeuroAD Discovery Engine (leakage probes, calibration, scoring, verdict, evidence ledger)

**Summary:** The pipeline is unusually careful for a hackathon: it uses StratifiedGroupKFold for site-disjoint outcome AUC (probe.py:167), out-of-fold probe scores for the biomarker anchor (gauntlet.py:219-231), a Fisher-z CI on the anchor correlation (gauntlet.py:198-216), Wilson and Cohen's-d CIs in cluster characterization (discovery.py:43-70), and bootstrap-Jaccard cluster stability (detective.py:136). The single biggest statistical gap is that every headline AUC (naive effect, leakage margin, replication) is a single-seed point estimate compared against hard thresholds with NO confidence interval and NO permutation/null baseline — so near-threshold verdicts (e.g. margin +0.04 -> WEAKENED) are within noise and not defensibly significant. A second real gap: the core probe runs logistic regression on the full 768-d Neuro-JEPA embedding with n as low as 61-96 (probe.py:100), so the builder-track referee's own AUCs on real cohorts are p>>n inflated and internally inconsistent with the researcher-track "honesty_note" that only trusts PCA-reduced numbers. No multiple-comparison control exists across the 5 tests x up-to-6 clusters, covariate residualization happens outside the CV loop, and "calibration.py" contains literature plausibility ranges, not any actual probability calibration. These are fixable and, if fixed, would sharply raise Depth and Demo-trust credibility.

**Strengths:**
- Subject/site-disjoint CV is real, not decorative: point_head emits site groups and cross_val_auc routes them into StratifiedGroupKFold for all outcome AUCs (probe.py:121-129, 165-173), with graceful fallback when groups are too few.
- The biomarker anchor is gated on the Fisher-z 95% CI LOWER BOUND of the correlation, not raw r, and uses out-of-fold probe scores (gauntlet.py:219-283) — this specifically defeats the 'lucky small-n correlation on noise passes' failure mode and is the most statistically mature part of the code.
- Honest self-flagging of p>>n: openbhb_neurojepa_leakage.json reports both the inflated raw-768d AUC (0.998) and the defensible PCA-10 AUC (0.931) with an explicit honesty_note — exactly the disclosure a reviewer wants for the researcher-track finding.
- Leakage-margin CV asymmetry (grouped outcome vs ungrouped scanner) is documented and argued to be a conservative lower bound (leakage.py:47-55), showing awareness of the direction of bias.
- Cluster discovery uses reduce-then-cluster with bootstrap-Jaccard stability as the primary gate and ARI/AMI vs planted truth (detective.py, discovery.py:226-232) — proper unsupervised rigor rather than eyeballing k.
- NA-renormalized scoring drops untestable dimensions from numerator AND denominator with an explicit completeness caveat (contract.py:165-183, scoring.py:110-114), so missing data cannot silently inflate the score.
- Determinism: RANDOM_STATE=0 pinned across probe, detective, PCA and CV, so runs are byte-reproducible.

**Findings:**
- [high] **No confidence intervals or permutation/null baseline on any headline AUC — verdicts are hard thresholds on single-seed point estimates** — Every gate that drives the verdict compares a single point-estimate AUC to a fixed cutoff with no uncertainty quantification: site_scanner uses margin<0.10 -> WEAKENED, margin<=0 -> FAILED (gauntlet.py:116-124); replication uses test_auc>=0.65 -> PASSED (gauntlet.py:321-326); retained-fraction bands 0.70/0.40 gate age_sex and brain_age (gauntlet.py:47-52). The shipped ledger shows margin +0.04 producing 'WEAKENED' (evidence_ledger.csv row 2) — a difference of two CV-AUCs that is almost certainly inside a 95% CI of zero. There is no DeLong test, no bootstrap AUC CI, and no label-permutation null anywhere in src/ (grep for delong/p_value/permut returns nothing but the anchor's Fisher-z). A rigorous reviewer will treat every near-threshold PASS/FAIL as a coin flip and discount the fragile/robust verdict.
    - evidence: gauntlet.py:116-124, 321-326, 47-52; reports/evidence_ledger.csv line 3 (margin +0.04 -> weakened); grep 'permut|delong|p_value' src/ -> none
- [high] **Core referee probe runs on full 768-d embeddings at n=61-96 (p>>n) — builder-track AUCs are inflated and inconsistent with the researcher-track honesty note** — LinearProbe fits LogisticRegression on contract.embedding_matrix(df) with NO dimensionality reduction (probe.py:47-58, 100). For the real Neuro-JEPA cohorts the embedding is 768-d while n_subjects is 61 (oasis_neurojepa_ad.json) to 96 (openbhb_neurojepa_leakage.json). The project itself acknowledges this exact regime is inflated ('Raw 768-d AUC (~0.998) is inflated by p>>n', openbhb_neurojepa_leakage.json honesty_note) and only trusts PCA-reduced numbers in the standalone scripts — but the gauntlet, scoring, and leakage_margin that produce the actual ClaimCards never apply that reduction. So the referee's own outcome/scanner/replication AUCs on any real high-dim cohort are optimistically biased, and the two tracks report methodologically different numbers for the same embeddings.
    - evidence: probe.py:100 (embedding_matrix, no PCA) + LogisticRegression C=1.0 probe.py:52-54; oasis_neurojepa_ad.json n=61; openbhb_neurojepa_leakage.json n=96 + honesty_note
- [medium] **Covariate and brain-age residualization is fit on the full dataset outside the CV loop** — test_age_sex and test_brain_age compute the adjustment by regressing covariates out of the ENTIRE embedding matrix once (_residualize fits StandardScaler + lstsq on all rows, gauntlet.py:55-63), then feed the residualized X into cross_val_auc. The covariate coefficients are therefore estimated using the same rows that appear in every test fold, so the 'after adjustment' AUC and the retained-fraction are not a clean nested-CV estimate. Brain-age partially guards this (OOF predictions for CN training rows, gauntlet.py:168-169) but the final _residualize(Xo, control) is still fit-and-applied on all outcome rows. The bias generally makes the 'after' AUC slightly optimistic-to-neutral, muddying the survivor/kill retained-fraction that the verdict hinges on.
    - evidence: gauntlet.py:55-63 (_residualize fit on full X), 94-96 (age_sex), 176-177 (brain_age)
- [medium] **No multiple-comparison control across 5 tests x up-to-6 clusters + biomarker and confound sub-tests** — discover_and_referee reruns the full 5-test gauntlet per cluster (k up to 6, discovery.py:195-224) plus 2 Cohen's-d biomarker CIs and 3 Cramer's-V/eta^2 confound associations per cluster, each with its own 95% CI or hard threshold, and none are FDR/Bonferroni-adjusted. The artifact-flag cutoff _ARTIFACT_ASSOC=0.25 (discovery.py:37) is asserted, not derived from a null distribution of Cramer's V under random cluster membership. With ~5x6 + characterizations, several 'significant' CIs excluding zero are expected by chance alone.
    - evidence: discovery.py:37 (_ARTIFACT_ASSOC=0.25 arbitrary), 180-224 (per-cluster gauntlet loop), 55-70 (_cohens_d CI), 139-153 (associations); no BH/Bonferroni in repo
- [medium] **Replication test can PASS on n_test as small as 6 with a single-fit AUC and no CI** — test_replication holds out the smallest usable site requiring only te.sum()>=6 (gauntlet.py:302-309), fits ONE LinearProbe, and computes a single held-out roc_auc_score with no CI (gauntlet.py:317-319), then gates test_auc>=0.65 -> PASSED. An AUC on 6 held-out subjects has a 95% CI spanning roughly [0.3,1.0]; a PASS there is essentially noise, yet replication carries 15 score weight and is the open-data corroboration path that unlocks promotion (scoring.py:87-89). The shipped ledger's n_test=62 is fine, but the code permits credit at n_test=6.
    - evidence: gauntlet.py:307 (te.sum()>=6), 317-319 (single-fit AUC, no CI), 321 (>=0.65 -> PASSED); scoring.py:87-89 (replication unlocks promotion)
- [medium] **'calibration.py' performs no probability calibration despite the pipeline consuming predict_proba downstream** — calibration.py contains hardcoded literature plausibility RANGES (CAL dict, lines 71-92) used as verdict thresholds — there is no reliability diagram, Brier score, or ECE anywhere. Yet raw predict_proba scores drive the biomarker anchor correlation (gauntlet.py:231) and the demo effect bars, and LinearProbe uses class_weight='balanced' (probe.py:54), which deliberately decouples output probabilities from base rates. The probabilities that flow into downstream statistics are never calibration-assessed, and the module name oversells what is really a citation/threshold table.
    - evidence: calibration.py:71-92 (ranges, not calibration); probe.py:54 (class_weight='balanced'); gauntlet.py:231 (proba used for anchor); grep 'brier|isotonic|reliability|ECE' -> none
- [medium] **Shipped evidence_ledger.csv is not self-describing or fully traceable — no provenance, and one effect value is mislabeled** — evidence_ledger.csv is written by app/build_demo_data.py:930-935 from the synthetic SURVIVOR case, but the CSV carries no dataset name, seed, source (engine vs calibrated-fallback), n, or CI columns — a reviewer who downloads it cannot tell it is synthetic (only cohort_card.json carries the 'SYNTHETIC HARNESS' badge, and it is a separate file). Worse, the biomarker_anchor row reports effect=0.904 labeled 'anchor strength |r|' while the same row's detail says 'r=+0.40': 0.904 is the UI transform 0.5+min(|r|,0.5) (build_demo_data.py:531), not |r|, so the shipped artifact's headline number contradicts its own detail text.
    - evidence: reports/evidence_ledger.csv line 5 (0.904 labeled |r| vs detail r=+0.40); app/build_demo_data.py:531 (_effect_bar transform), 930-935 (writer, no provenance cols)
- [low] **Single fixed seed with no seed-sensitivity / repeated-CV assessment near thresholds** — All CV uses shuffle=True, random_state=0 (probe.py:167-172) and RANDOM_STATE=0 is pinned everywhere. This is great for reproducibility but means no verdict-stability check: for cases sitting near the 0.10 margin or 0.70/0.40 retained bands, a different seed could flip WEAKENED<->PASSED<->FAILED. The code comments claim seed-robustness only for the anchor (gauntlet.py:203); the AUC-threshold verdicts have no analogous defense. No repeated k-fold or seed-sweep is reported.
    - evidence: probe.py:167-172 (single random_state=0), gauntlet.py:203 (seed-robustness claimed only for anchor)

**Enhancements:**
- [M] **Add bootstrap/DeLong AUC CIs + a label-permutation null p-value to every headline metric** — For naive effect, outcome_auc, scanner_auc, leakage margin, and replication AUC, add a bootstrap over subjects (or DeLong for paired AUCs) to report 95% CIs, plus a stratified label-permutation null (e.g. 1000 shuffles, honoring the site groups) to report p vs chance and p vs scanner. Then reframe thresholds as 'CI excludes chance' / 'margin CI excludes zero' rather than point-estimate cutoffs. Highest-ROI change: it directly answers the reviewer's first question and upgrades every verdict from an assertion to a test.
- [M] **Reconcile builder- and researcher-track dimensionality: reduce inside the probe for p>>n cohorts** — Add an optional PCA/whitening (or strong-ridge) front-end to LinearProbe, selected automatically when n < k*D, and make the gauntlet + leakage_margin use the SAME reduction the standalone neurojepa scripts already trust. Report effective degrees of freedom. This removes the internal inconsistency where the honesty_note distrusts the very AUCs the engine ships.
- [M] **Move covariate/brain-age adjustment inside the CV fold (nested residualization)** — Wrap StandardScaler + covariate-regression + logistic head in one sklearn Pipeline (or a custom transformer) so the residualization is fit only on each training fold, and report retained-fraction with a bootstrap CI. Eliminates the out-of-fold-adjustment leakage in age_sex and brain_age and makes 'effect retained' a defensible nested-CV estimate.
- [M] **Add Benjamini-Hochberg FDR across gauntlet tests and clusters, and derive the artifact threshold from a null** — Collect all per-test / per-cluster p-values (from the permutation nulls above and the anchor CI) and apply BH-FDR, reporting q-values on the ClaimCard and cluster cards. Replace the hardcoded _ARTIFACT_ASSOC=0.25 with a permutation null of Cramer's V under shuffled cluster membership (flag when observed exceeds the 95th percentile). Closes the multiplicity critique on the RESEARCHER-track discovery output.
- [S] **Make evidence_ledger.csv self-describing and reproducible** — Emit provenance columns/header: dataset, seed, source (engine|fallback), n_used, n_missing, effect, ci_lo, ci_hi, p_perm, plus a top comment line with contract version and a 'SYNTHETIC HARNESS' flag matching cohort_card. Fix the biomarker_anchor row so the 'effect' column and its label are consistent (report r with its CI, not the 0.5+|r| UI transform, or relabel the column 'effect bar'). A downloaded ledger should stand alone as an audit artifact.
- [S] **Add a seed-sweep verdict-stability panel** — Run the referee across ~20 seeds on the synthetic SURVIVOR and KILL and report the distribution of scores and the verdict-flip rate near band boundaries; surface it in the demo as 'verdict stable across 20 seeds' (or honestly widen the MIXED band where it flips). Turns the single-seed determinism from a hidden risk into a demonstrated robustness claim.
- [S] **Add a genuine calibration check for the probabilities used downstream** — Since predict_proba scores feed the anchor correlation and effect bars, add an OOF reliability diagram + Brier score / ECE for the probe (at least on the synthetic and OASIS cohorts) and either report it or switch to CalibratedClassifierCV. At minimum rename/annotate calibration.py so the module name matches its contents (plausibility ranges, not probability calibration).

## Engineering / Architecture

**Summary:** Well-architected Python package: a frozen contract layer (contract.py) cleanly decouples data feeders from the referee engine, the "run in one command" claim genuinely holds (both `neuroad demo` via -m and the installed `neuroad` console script work, producing the compelling 88/100 SURVIVOR vs 15/100 KILL punchline), and 75/75 pytest tests pass in ~25s with determinism pinned via RANDOM_STATE. The Claude layer is a real strength for the Claude Use score: five persona modules plus a three-voice courtroom, strict structured tool-use, a server-side-refusal beta + retry-model chain, and rich deterministic fallbacks on every call. The main gaps are reproducibility and demo-verifiability: the real Neuro-JEPA embedding tables behind the researcher-track finding are git-ignored (so a clean clone cannot regenerate the headline leakage result), and by default every Claude call silently degrades to offline templates with no live/offline indicator, leaving the "Claude as adjudicator" story unverified in the shipped demo.

**Strengths:**
- contract.py is a genuine frozen-interface seam: METADATA_COLUMNS + validate_table + one linear head pointed at LABEL_TARGETS means swapping real embeddings changes the table, not the code. Clean, modular, data-source-independent.
- One-command claim holds: installed `neuroad` console script and `python -m neuroad.cli demo` both run fully offline and produce the SURVIVOR-promotes / higher-AUC-KILL-refused punchline (contract-level demo integrity).
- 75/75 tests pass (tests/test_*.py) covering engine, pipeline, data, gated mapper, discovery, claude layer; determinism handled via probe.RANDOM_STATE and CI-lower-bound gauntlet gates.
- Sophisticated, creative Claude use: claim_parser + courtroom(3 voices) + bridge + reviewer + narrator, strict single-tool structured output (_client.py:90-97), server-side refusal fallback beta + sonnet retry chain, each with a rich deterministic fallback so the pipeline never crashes offline.
- Strong secrets / gated-weight hygiene: .gitignore blocks *.safetensors/*.pt/*embeddings*.csv/.env; gated.py is a clean drop-in seam mapping raw exports to the contract with zero downstream change; compliance reasoning documented in loader docstrings.
- skills/ are real Agent Skills (SKILL.md YAML frontmatter) whose run.py call the actual neuroad.gauntlet functions — composable entrypoints, not duplicated logic.

**Findings:**
- [high] **Researcher-track finding is not reproducible from a clean clone (embedding tables git-ignored)** — The .gitignore rule `*embeddings*.csv` excludes data/real/oasis1_neurojepa_embeddings.csv and openbhb_neurojepa_embeddings.csv (confirmed via git check-ignore), so loaders.load('openbhb:neurojepa') / 'oasis:neurojepa' raise FileNotFoundError on a fresh clone. The headline scanner-leakage result (AUC 0.93 on frozen Neuro-JEPA) survives only as committed reports/*.json plus a Colab notebook, not as a runnable path.
    - evidence: data/real/oasis_jepa.py, openbhb_jepa.py load these CSVs; .gitignore:*embeddings*.csv; git check-ignore confirms both ignored; git ls-files shows neither tracked
- [medium] **Default demo runs 100% offline templates with no live-Claude indicator** — USING_LIVE_API is False without ANTHROPIC_API_KEY, so claim_parser/courtroom/bridge/reviewer/narrator all silently return deterministic fallbacks; nothing in CLI output or reports states whether Claude actually ran. The pipeline shims (pipeline.py:32-92) also swallow every exception with bare except:pass, so a misconfigured live key degrades invisibly.
    - evidence: _client.py:38,63-69; pipeline.py:39,57,67,80 bare `except Exception: pass`; cli._fmt_card prints no live/offline badge
- [medium] **UI does not read the reports `neuroad demo` writes — separate build step, risk of stale demo** — README quickstart says `open app/index.html` 'ticks the gauntlet through reports/*.json', but index.html fetch()es app/demo_data.json (built only by app/build_demo_data.py) and on the file:// path CORS blocks fetch, falling back to embedded JSON. Running `neuroad demo` regenerates reports/ that the UI never consumes, and demo_data.json is a committed 87KB artifact that can drift from the engine.
    - evidence: app/index.html:4350 fetch('demo_data.json'); build_demo_data.py writes app/demo_data.json + reports/; cli._cmd_demo writes only reports/
- [low] **Packaging/config inconsistencies undercut open-source polish** — requirements.txt hardcodes anthropic even though pyproject marks it an optional [claude] extra; the egg-info is a stub (only dependency_links.txt); and gated.py's module docstring (lines 32-35) claims it is 'intentionally NOT wired into loaders.py' while loaders.py:48-51 in fact dispatches gated datasets. No .github CI exists to show reviewers a green build.
    - evidence: requirements.txt vs pyproject [project.optional-dependencies].claude; src/neuroad_referee.egg-info/ has only dependency_links.txt; gated.py:32-35 vs loaders.py:48-51; no .github/

## Design / UI-UX (single-file HTML workbench, app/index.html)

**Summary:** NeuroAD's workbench (app/index.html, ~4360 lines, all CSS/JS inline, offline-safe) is already a genuinely sophisticated, credible "scientific referee" UI: a dark 3-column workbench with masthead + substrate toggle, cohort card, docket, PCA canvas scatter with 1σ ellipses/centroids, a staged-reveal gauntlet with eased effect-size bars, an animated verdict-meter needle, a Claude courtroom + self-critiquing reviewer, a confound leaderboard, KILL-vs-SURVIVOR split, and an export tray — with deterministic data, progressive disclosure, and prefers-reduced-motion support. The engineering craft is real and it reads as a tool a scientist would trust, which serves Impact and Depth well. But for Demo (30% of score) it leaves its best beats on the table: the two hero numbers snap instead of counting up, the scatter hard-swaps instead of morphing when it should make leakage visible, the verdict lands without a stamp, and there is no spotlight/autopilot to guide the eye or guarantee a repeatable take. Legibility is also a video risk — pervasive 10–11px mono microtext and low-contrast grays will turn to mush at 1080p/phone. The fixes are almost all cheap, single-file, and disproportionately raise the "cool to watch" ceiling without touching the science.

**Strengths:**
- Strong, coherent visual identity: dark instrument-panel aesthetic, tabular-mono numerics, consistent semantic color (green=survives, red=kill, amber=weakened, star=★ decisive tests) — reads instantly as a serious lab tool, not a toy app.
- Real motion foundations already in place: staged gauntlet reveal (680ms/step at index.html:4289-4293), CSS-eased bar fills with cubic-bezier(.2,.8,.2,1) (line 103, 186), animated meter needle (line 207), staggered court/reviewer li fade-ins, and a pulsing 'running' status dot — the choreography scaffolding exists to build on.
- The scatter is conceptually the wow moment: seeded deterministic points + per-class 1σ ellipses + ringed centroids (drawScatter, 3632-3683) make the 'color-by-scanner shows the machine IS the axis' finding literally visible — the core researcher-track insight is already spatial.
- Genuinely novel narrative devices that photograph well on video: the KILL-vs-SURVIVOR split modal, the leakage-margin pair-bar (outcome AUC vs scanner AUC on a shared axis), the gated biology zone (🔒 locked until promoted), and Claude arguing prosecution/defense/judge THEN critiquing its own verdict — this is strong Claude-Use material.
- Disciplined craft: offline/file:// safe with embedded-JSON fallback, prefers-reduced-motion honored with an instant-render path (4280-4287), keyboard controls, ARIA roles/labels, focus-visible outlines, progressive disclosure via collapsible panels — this is refined, not a hack.

**Findings:**
- [high] **Hero numbers snap instead of animating — the single cheapest 'wow' is missing** — The 34px verdict score and the leakage-margin headline are set via textContent (renderVerdict: vScore.textContent=c.score at 3873; mv.textContent at 3861) with no count-up/roll. The needle glides (CSS transition, line 207) but the number it points to just appears. On a 3-min video the verdict is THE payoff beat (FRAGILE 15 vs STRONG CANDIDATE 88) and it lands flat. A 900ms eased count-up synced to the needle glide is the highest-ROI polish in the whole app.
    - evidence: app/index.html:3873, 3861, 3826 (naive AUC big number also static)
- [high] **Scatter hard-swaps on color toggle — the leakage finding is not made cinematic** — toggleColor just calls drawScatter() which fully clears and redraws (3632-3637, 4311-4313). Switching color:outcome→color:scanner on the KILL is THE researcher-track reveal ('the dominant axis of variance is the scanner, not the disease') but it happens as an instantaneous repaint — no tween, no ellipse rotation, no centroids sliding apart. Points are re-grouped and ellipses recomputed each call, so the machinery to animate a per-point morph + ellipse re-orientation is already there; it just needs a requestAnimationFrame interpolation between the two colorings.
    - evidence: app/index.html:3632 (drawScatter clears+redraws), 4311-4313 (toggleColor→drawScatter, no tween)
- [high] **No autopilot / guided demo mode — the 3-min take depends on live human input** — The demo is driven entirely by manual Space/←→/C/S/V keypresses (bind, 4334-4343). DEMO_SCRIPT.md specifies a precise beat-by-beat timeline (cold-open KILL → star test → FRAGILE → flip SURVIVOR → biomarker rescue → courtroom → real OASIS), but nothing in the app executes it. Recording a clean, identically-timed 3-min take requires flawless manual choreography every attempt — fragile for the single most-weighted deliverable. A scripted timeline array driving the existing selectCase/runGauntlet/toggleColor/toggleSubstrate/openSplit functions with timed lower-third captions would make every take deterministic.
    - evidence: app/index.html:4334-4343 (only manual keybinds); docs/DEMO_SCRIPT.md (a timeline that has no code counterpart)
- [high] **Video legibility: pervasive 10–11px mono microtext + low-contrast grays** — Core labels run 10–11.5px mono on near-black: keyhints 10.5px (line 46), panel tags 10px (line 81), gauntlet weights/axes 10-11px (173, 188-189), src-chip 10.5px, scatter axis labels 11px canvas text (3680). Support inks --mute #6a7d92 and --ink3 #8ea3b8 on --bg #0b0f14 sit near/below WCAG AA. At 1080p H.264 compression and phone viewing these turn to mush while only the hero numbers survive — so the supporting evidence (the AUC values, axis domains, chips that make the tool credible) becomes unreadable exactly when a judge is deciding whether to trust it.
    - evidence: app/index.html:46, 81, 173, 188-189, 3680; palette --mute/--ink3/--bg at lines 6-9
- [medium] **No spotlight/focus — dense 3-column layout gives the eye no guide on video** — Three columns + a full-width extras section (real-data + Detective) + a fixed export tray are all fully lit simultaneously (.wrap grid, 70-72; #extras, 557). During each narrative beat nothing dims or elevates the active panel, so a viewer watching the video has no cue where to look as the story moves docket→scatter→gauntlet→verdict→courtroom. High information density is a strength for a working scientist but a liability for a passive 3-min viewer.
    - evidence: app/index.html:70-72 (.wrap), 557 (#extras), no active-panel highlight class anywhere in render path
- [medium] **Verdict lands without a stamp — the biggest emotional beat is undramatized** — On completion the promo chip is simply shown (vp.style.display='inline-flex', 3878-3879) and the panel keeps its neutral chrome. There's a small .flash on gauntlet rows (line 402) but the verdict panel itself doesn't react — no rubber-stamp slam, no full-panel red/green color flood distinguishing REFUSED from PROMOTED. The refuse-vs-promote contrast is the whole thesis and it currently reads as a quiet text change.
    - evidence: app/index.html:3877-3879 (promo chip appears), 219-224 (.promo static), 402 (flash only on rows)
- [medium] **Boot default state contradicts the demo script's cold-open** — State initializes to S={sub:'oasis', kase:'SURVIVOR'} (line 3609), but DEMO_SCRIPT.md §0:00 opens 'with the KILL cohort loaded' on the SYNTHETIC harness (the KILL naive AUC 0.82 / scanner 0.95 / margin −0.13 / score 15 live in the synthetic substrate, not OASIS). Every recording therefore requires manual pre-setup (press S, then select KILL) before rolling — an easy way to fumble a take or record the wrong opening frame. A URL-param entry state (?sub=synthetic&case=KILL) or matching the default to the script start removes the footgun.
    - evidence: app/index.html:3609 (default oasis/SURVIVOR) vs docs/DEMO_SCRIPT.md:8-14 (cold-open KILL on synthetic)
- [medium] **Strongest Claude-Use beat (courtroom + self-critique) is collapsed by default** — The Reviewer panel ships with class 'collapsible collapsed' (line 545) and the Courtroom is collapsible (505). runGauntlet populates them (renderReviewer/renderCourt at endT+300/+600, 4301-4302) but does NOT auto-expand them, so 'Claude argues both sides then argues against its own verdict' — arguably the single best Claude-Use narrative in the project — is hidden behind a chevron unless the presenter remembers to open it on camera.
    - evidence: app/index.html:545 (reviewer starts collapsed), 4301-4302 (populated but not expanded on run)
- [low] **Leakage pair-bar renders at once instead of choreographing the 'scanner catches up' race** — pairBarHTML builds outcome/scanner/margin bars in one innerHTML write (3842-3864); they animate only via the generic CSS width transition, both at once.     - evidence: app/index.html:3842-3864 (single-shot render), 413-420 (.pb-fill CSS transition only)
